1. Main User Flow (Buy Button Click)

flowchart TD
    A["User clicks BUY"] --> B["GET /api/orders/pending?key_name=xxx"]

    C -- No --> D["POST /api/orders\n(status = PREPARED)"]
    D --> E["Show Payment Method Selector"]
    E --> F{User picks method}
    F -- Stripe --> G["POST /api/orders/:id/pay\n{method: stripe}"]
    F -- PayPay --> H["POST /api/orders/:id/pay\n{method: paypay}"]
    G --> I["Redirect to Stripe Checkout"]
    H --> J["Redirect to PayPay QR"]

    C -- Yes --> K["Show PendingOrderDialog"]
    K --> L{User choice}

    P -- Yes --> E
    P -- "No (409)" --> Q["Show: Payment is\nbeing processed"]

    I --> R["User pays"]
    J --> R
    R --> S["Webhook from Provider"]
    S --> T["Order → SUCCESS\nPermission created"]
    T --> U["/payment/:orderId detects\nstatus change via polling"]
    U --> V["Redirect to /paid\n(success / gacha page)"]

2. Order State Machine

stateDiagram-v2
    [*] --> PREPARED : Order created

    PREPARED --> PENDING : Payment session created
    PREPARED --> CANCELLED : User cancels before paying

    PENDING --> SUCCESS : Webhook confirms payment
    PENDING --> FAILED : Webhook: payment failed/expired
    PENDING --> PREPARED : User cancels to switch method
    PENDING --> CANCELLED : User fully cancels order

    CANCELLED --> SUCCESS : Race condition:\nwebhook after cancel\n(honor payment)

    SUCCESS --> [*]
    FAILED --> [*]

3. Konbini Payment & Recovery Flow

sequenceDiagram
    actor User
    participant FE as Frontend
    participant BE as Backend
    participant Stripe

    User->>FE: Click "Buy"
    FE->>BE: POST /api/orders
    BE-->>FE: order #501 (PREPARED)
    FE->>BE: POST /api/orders/501/pay {stripe}
    BE->>Stripe: Create Checkout Session
    Stripe-->>BE: session URL
    BE-->>FE: redirect_url
    FE->>Stripe: Redirect to Checkout
    User->>Stripe: Picks Konbini, sees voucher
    Stripe->>BE: Webhook: session.completed (payment_status=unpaid)
    BE->>BE: Store hosted_voucher_url + expires_at in payment_metadata
    Note over User: User closes browser, forgets code

    rect rgb(255, 243, 224)
      Note over User,Stripe: Next day - User returns
      User->>FE: Click "Buy gl-3" again
      FE->>BE: GET /api/orders/pending?key_name=gl-3
      BE-->>FE: Order #501 (konbini, expires Apr 18)
      FE->>User: PendingOrderDialog: "Pending konbini payment"
      User->>FE: Click "View Payment Instructions"
      FE->>User: Opens hosted_voucher_url (payment code visible)
    end

    User->>Stripe: Pays at convenience store
    Stripe->>BE: Webhook: async_payment_succeeded
    BE->>BE: Order #501 → SUCCESS, create Permission
    FE->>BE: Polling detects SUCCESS
    FE->>User: Redirect to /paid

4. Cancel Payment & Switch Method

sequenceDiagram
    actor User
    participant FE as Frontend
    participant BE as Backend
    participant PayPay
    participant Stripe

    User->>FE: Click "Buy"
    FE->>BE: POST /api/orders → order #502
    FE->>BE: POST /api/orders/502/pay {paypay}
    BE->>PayPay: Create QR
    PayPay-->>BE: QR URL
    BE-->>FE: redirect to PayPay

    Note over User: User changes mind, wants card

    User->>FE: Click "Cancel & Choose New Method"
    FE->>BE: POST /api/orders/502/cancel-payment?full_cancel=false
    BE->>PayPay: cancelPayment() API
    PayPay-->>BE: OK
    BE->>BE: Order → PREPARED, clear invoice_id
    BE-->>FE: 200 OK

    FE->>User: Show Payment Method Selector
    User->>FE: Picks "Stripe"
    FE->>BE: POST /api/orders/502/pay {stripe}
    BE->>Stripe: Create Checkout Session
    Stripe-->>BE: session URL
    FE->>Stripe: Redirect
    User->>Stripe: Pays with card
    Stripe->>BE: Webhook → SUCCESS
    BE->>BE: Permission created
